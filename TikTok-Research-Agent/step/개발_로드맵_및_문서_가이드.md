## 🔗 논리적 연결 구조

### 1️⃣ docs/ 폴더: "5대 질문" 설계 시스템

각 문서는 **하나의 핵심 질문**에 답합니다.

```
[사용자 질문] "틱톡 리서치 에이전트를 만들고 싶어요"
        ↓
[Step 1] Ideation.md 읽기
        → "왜 이걸 만들어야 하는가?" 이해
        → 삼양식품 채용 요건, 비즈니스 가치 파악
        ↓
[Step 2] PRD.md 읽기
        → "정확히 무엇을 만들 것인가?" 명확화
        → Must-have vs Should-have 구분
        → 성공 기준 설정
        ↓
[Step 3] Tasks.md 읽기
        → "7일 동안 매일 무엇을 할 것인가?" 계획
        → Day 1-2: Agent 1, Day 3-4: Agent 2...
        ↓
[Step 4] TechStack.md 읽기
        → "어떤 기술로 구현할 것인가?" 결정
        → n8n, Claude API, Airtable, Python
        ↓
[Step 5] Tutorial.md 읽기
        → "다른 사람도 따라 만들 수 있는가?" 검증
        → 10단계 실습 가이드
```

#### 📊 5대 질문 매핑 테이블

| 문서 | 핵심 질문 | 독자 | 읽는 시점 | 다음 행동 |
|------|----------|------|-----------|-----------|
| **Ideation.md** | Why? 왜? | 나 자신, 면접관 | 프로젝트 시작 전 | 동기 부여, 방향 설정 |
| **PRD.md** | What? 무엇? | 개발자, PM | 설계 시작 시 | 요구사항 이해, 범위 확정 |
| **Tasks.md** | When? 언제? | 나 자신, PM | 개발 일정 수립 시 | 일별 작업 체크리스트 |
| **TechStack.md** | How? 어떻게? | 개발자, 기술 리뷰어 | 기술 선택 시 | 구현 시작 |
| **Tutorial.md** | Follow? 따라하기? | 신규 사용자 | 완성 후 배포 시 | 실제 사용/재현 |

### 2️⃣ .claude/ 폴더: "개발 자동화 매뉴얼"

Claude Code가 **효율적으로 개발하도록 돕는 지식 저장소**입니다.

```
[개발 시작]
        ↓
[Claude Code 실행]
        ↓
[skill-rules.json 로드]
        → "에이전트 개발" 키워드 감지
        → agent-dev-guidelines skill 자동 활성화
        ↓
[SKILL.md 참조]
        → 맥도날드 시스템 원칙 적용
        → 7단계 워크플로우 따름
        → CLAUDE.md 200줄 엄수
        ↓
[WORKFLOWS.md 참조]
        → n8n 워크플로우 패턴 복사
        → Agent 1/2/3 구조 적용
        ↓
[코드 작성]
        ↓
[세션 종료]
        ↓
[update-tutorial.sh 자동 실행]
        → Tutorial.md에 세션 로그 추가
```

#### 🧠 Skills 별 역할

| Skill | 언제 활성화? | 제공하는 것 | 예시 |
|-------|------------|-----------|------|
| **agent-dev-guidelines** | "n8n", "에이전트" 입력 시 | 바이브코딩 패턴, 아키텍처 | "3개 독립 에이전트 구조로 설계" |
| **prompt-engineering** | "프롬프트", "LLM" 입력 시 | 환각 방지, RAG 프롬프트 | "데이터 없으면 '모름' 답변" |
| **tutorial-generator** | "튜토리얼" 입력 시 | 문서 템플릿, 추출 패턴 | "Step 1: 환경 설정 (20분)" |
| **mcp-server-dev** | "MCP" 입력 시 | MCP 서버 구현 예제 | "Airtable → MCP 프로토콜 변환" |

## 💡 핵심 인사이트

### "설계도가 완벽하면, 집 짓기는 쉽다"

| 비유 | 현재 단계 (Phase 1) | 다음 단계 (Phase 2) |
|------|-------------------|-------------------|
| **건축** | 설계도면 완성 | 실제 시공 |
| **요리** | 레시피 작성 | 재료로 요리 |
| **여행** | 여행 일정표 작성 | 실제 여행 |
| **코딩** | 문서화 완료 ✅ | 코드 구현 예정 |

### 바이브코딩의 지혜

1. **"PRD부터 쓴다"**
   - ❌ 코드부터 작성 → 방향 잃음
   - ✅ PRD 먼저 → 코드는 자동으로 따라옴

2. **"작동하는 것을 먼저 만든다"**
   - Phase 1에서 완벽한 계획 수립
   - Phase 2에서 빠르게 프로토타입 완성 (1주일)
   - Phase 3에서 개선 및 최적화

3. **"200줄 CLAUDE.md"**
   - 핵심만 남기고 나머지는 docs/ 분리
   - Claude Code가 필요할 때만 skills/ 로드
   - 토큰 절약 = 비용 절약
